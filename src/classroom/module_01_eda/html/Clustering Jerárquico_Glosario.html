1

Clustering jerárquico

Base

El clustering jerárquico es un método de aprendizaje no supervisado que agrupa observaciones según su similitud y construye una estructura en forma de árbol. A diferencia de K-Means, no parte de una cantidad fija de clusters, sino que permite observar cómo los datos se van agrupando progresivamente.

En la versión más usada, llamada aglomerativa, cada observación comienza como un cluster individual. Luego, el algoritmo va fusionando los puntos o grupos más parecidos hasta que todos quedan integrados en una única estructura jerárquica. 

Inicio: cada punto es un cluster
Proceso: fusionar grupos parecidos
Resultado: dendrograma o árbol jerárquico
2

Aprendizaje no supervisado

Base

El aprendizaje no supervisado trabaja con datos que no tienen etiquetas previas. No existe una variable objetivo que indique cuál es la respuesta correcta. El algoritmo debe encontrar patrones, asociaciones o estructuras ocultas a partir de las variables disponibles.

En clustering jerárquico, esto significa que el algoritmo no sabe previamente qué observaciones pertenecen a cada grupo. Solamente mide similitudes o distancias y construye una organización progresiva.

3

Observación, punto o dato

Base

Una observación es cada unidad del dataset. Puede representar un cliente, un libro, una empresa, un país, una película, una transacción o un documento. Matemáticamente, cada observación se representa como un punto en un espacio de variables.

Si usamos dos variables, el punto puede visualizarse en un plano. Si usamos muchas variables, el punto vive en un espacio de mayor dimensión, aunque ya no sea fácil de dibujar. 

Cliente A = (edad, ingreso, gasto mensual)
Libro B = (páginas, dificultad, vocabulario técnico)
4

Cluster

Base

Un cluster es un grupo de observaciones que se consideran parecidas entre sí según las variables utilizadas y la medida de distancia elegida. En clustering jerárquico, los clusters se forman mediante fusiones.

Al inicio, cada observación es un cluster de tamaño uno. Luego se forman clusters de dos puntos, después de tres o más, hasta llegar a clusters más grandes. 

Inicio:
A | B | C | D

Después:
(A,B) | C | D

Más adelante:
(A,B,C) | D
5

Distancia

Base

La distancia mide qué tan diferentes son dos observaciones. Una distancia baja indica que dos puntos son parecidos; una distancia alta indica que son diferentes. El clustering jerárquico depende directamente de estas distancias.

Distancia baja → mayor similitud
Distancia alta → mayor diferencia

La distancia más frecuente para datos numéricos es la distancia euclídea, aunque también pueden usarse otras, como Manhattan o coseno, dependiendo del problema.

6

Matriz o vector de distancias

Proceso

Antes de realizar fusiones, el algoritmo necesita calcular las distancias entre pares de observaciones. Si hay n observaciones, la cantidad de distancias distintas entre pares es:

n × (n - 1) / 2

Por ejemplo, si hay 120 observaciones: 

120 × 119 / 2 = 7140 distancias

Estas distancias permiten saber qué puntos o grupos son candidatos naturales a fusionarse primero.

7

Fusión

Proceso

Una fusión es el momento en que el algoritmo une dos elementos para formar un nuevo cluster. Esos elementos pueden ser dos puntos individuales, un punto y un cluster, o dos clusters ya formados. 

Punto + punto → cluster nuevo
Punto + cluster → cluster más grande
Cluster + cluster → cluster superior

Cada fusión reduce en uno la cantidad total de clusters. Por eso, si empezamos con n observaciones, se necesitan n - 1 fusiones para terminar con un único árbol.

8

Linkage o criterio de enlace

Proceso

El linkage define cómo se mide la distancia entre clusters cuando los grupos ya contienen más de una observación. Esta elección es fundamental porque puede cambiar el dendrograma y la interpretación final.

Método Pregunta que responde
Single ¿Cuál es la menor distancia entre puntos de ambos clusters?
Complete ¿Cuál es la mayor distancia entre puntos de ambos clusters?
Average ¿Cuál es la distancia promedio entre los puntos de ambos clusters?
Ward ¿Qué fusión aumenta menos la dispersión interna?
9

Single linkage

Proceso

Single linkage mide la distancia entre dos clusters usando la menor distancia entre cualquier par de puntos, uno de cada cluster. Es decir, si dos grupos tienen aunque sea dos puntos muy cercanos, pueden fusionarse.

Su ventaja es que puede detectar estructuras alargadas o conectadas. Su desventaja es que puede producir el efecto cadena, donde puntos o grupos se van uniendo por conexiones locales aunque el grupo total quede poco compacto. 

Distancia single(A,B) = mínima distancia entre puntos de A y puntos de B
10

Complete linkage

Proceso

Complete linkage mide la distancia entre dos clusters usando la mayor distancia entre puntos de ambos grupos. Es un criterio más exigente porque solo fusiona grupos si incluso sus puntos más alejados no están demasiado separados.

Tiende a producir clusters más compactos que single linkage, aunque puede ser sensible a puntos extremos. 

Distancia complete(A,B) = máxima distancia entre puntos de A y puntos de B
11

Average linkage

Proceso

Average linkage calcula la distancia entre dos clusters como el promedio de todas las distancias entre puntos de un grupo y puntos del otro. Es una alternativa intermedia entre single y complete linkage.

Suele producir soluciones más equilibradas, porque no se basa solamente en el par más cercano ni en el par más lejano. 

Distancia average(A,B) = promedio de todas las distancias entre A y B
12

Método Ward

Proceso

El método Ward elige en cada paso la fusión que produce el menor aumento de la dispersión interna del nuevo cluster. Su objetivo es formar grupos compactos y relativamente equilibrados.

Ward suele funcionar muy bien cuando las variables son numéricas, están estandarizadas y se usa distancia euclídea. Es uno de los métodos más utilizados para obtener dendrogramas interpretables.

Ward → fusionar los grupos que menos aumentan la variabilidad interna
13

Matriz linkage

Proceso

La matriz linkage es la tabla que registra todas las fusiones realizadas por el algoritmo. Cada fila representa una fusión y contiene cuatro datos principales: los dos elementos fusionados, la distancia de fusión y el tamaño del nuevo cluster. 

[cluster_1, cluster_2, distancia_de_fusión, cantidad_de_observaciones]

Ejemplo: 

[107, 114, 0.00228, 2]

Esto significa que la observación 107 se fusionó con la observación 114 a una distancia de 0.00228 y formaron un cluster con 2 observaciones.

14

Dendrograma

Evaluación

El dendrograma es el gráfico en forma de árbol que representa visualmente las fusiones del clustering jerárquico. Abajo aparecen las observaciones o clusters pequeños; hacia arriba se ven las fusiones entre grupos cada vez más grandes.

La altura de cada unión indica la distancia o disimilitud entre los grupos fusionados. Por eso, una unión baja representa elementos muy parecidos y una unión alta representa grupos más diferentes.

Dendrograma = representación visual de la matriz linkage
15

Altura de fusión

Evaluación

La altura de fusión es el valor vertical del dendrograma donde dos puntos o grupos se unen. Representa la distancia o disimilitud entre los elementos fusionados. 

Fusión baja → grupos muy parecidos
Fusión alta → grupos más diferentes

Las alturas grandes son especialmente importantes porque suelen indicar que el algoritmo está empezando a unir grupos que antes estaban bastante separados.

16

Corte del árbol

Evaluación

El corte del árbol es la decisión de trazar una línea horizontal sobre el dendrograma para obtener una cantidad concreta de clusters. Cada rama que queda separada por debajo de esa línea representa un cluster final. 

Corte bajo  → muchos clusters pequeños
Corte medio → clusters principales
Corte alto  → pocos clusters grandes

Una estrategia habitual es cortar antes de las fusiones muy altas, porque esas fusiones suelen indicar la unión de grupos bastante diferentes.

17

Estandarización

Cuidados

La estandarización consiste en transformar las variables para que sean comparables. Es muy importante porque el clustering jerárquico depende de distancias. Si una variable tiene valores mucho más grandes que otra, puede dominar el cálculo de distancia.

z = (x - media) / desviación estándar

Por ejemplo, si usamos edad e ingreso, el ingreso puede dominar el análisis porque sus valores numéricos son mucho mayores. Estandarizar ayuda a evitar esa distorsión.

18

Limitaciones del clustering jerárquico

Cuidados

El clustering jerárquico es muy útil para explorar datos e interpretar estructuras, pero tiene limitaciones. Puede ser costoso con datasets grandes, sensible a outliers y dependiente de la escala de las variables.

  • Puede ser lento cuando hay muchas observaciones.
  • Los outliers pueden generar ramas aisladas o distorsionar el dendrograma.
  • El resultado cambia según la distancia y el método de linkage elegidos.
  • Si las variables no están escaladas, una variable puede dominar el análisis.

Por eso conviene revisar los datos, escalar variables, analizar outliers y comparar distintos métodos de linkage.
