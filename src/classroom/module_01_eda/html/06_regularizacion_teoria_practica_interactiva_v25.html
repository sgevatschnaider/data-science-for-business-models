1. Idea central: el error bajo no siempre significa buen aprendizaje

Entrenar no es memorizar

Una red neuronal puede reducir mucho el error de entrenamiento aprendiendo detalles accidentales del conjunto de datos. Eso puede parecer éxito durante el entrenamiento, pero falla cuando aparecen datos nuevos.

Señal típica de sobreajuste: la pérdida de entrenamiento baja, mientras la pérdida de validación deja de bajar o empieza a subir.

Regularizar es restringir inteligentemente

Regularizar significa introducir una presión para que el modelo prefiera soluciones más simples, más estables y menos dependientes del ruido. No busca empeorar el modelo; busca mejorar su capacidad de generalizar.

Meta: no obtener el menor error posible en train, sino el mejor desempeño posible en datos no vistos.

2. Técnicas principales de regularización

L2 / Weight Decay

L2 agrega una penalización proporcional al cuadrado de los pesos. En términos intuitivos, le dice al modelo: “podés aprender, pero evitá pesos demasiado grandes”. Los pesos grandes suelen hacer que la función sea muy sensible a pequeñas variaciones de entrada.

Loss_total = Loss_datos + λ · Σ w²

Efecto visual: la curva aprendida se vuelve más suave. Cuándo usarla: como regularizador general por defecto en redes densas o convolucionales. Riesgo: un λ excesivo produce underfitting.

L1 / Sparsity

L1 penaliza la suma de valores absolutos de los pesos. A diferencia de L2, tiende a empujar algunos pesos exactamente hacia cero. Esto favorece modelos más escasos, donde algunas variables o conexiones dejan de participar.

Loss_total = Loss_datos + λ · Σ |w|

Efecto didáctico: actúa como selección de variables o selección de conexiones. Cuándo usarla: cuando se busca interpretabilidad o reducir variables irrelevantes. Riesgo: demasiada L1 puede eliminar información útil.

Dropout

Dropout apaga aleatoriamente un porcentaje de neuronas durante el entrenamiento. Esto obliga a la red a no depender de una única ruta interna y a construir representaciones más robustas.

Durante entrenamiento: algunas activaciones se ponen en cero.
Durante inferencia: se usa la red completa ajustando la escala.

Analogía: entrenar un equipo donde algunos jugadores faltan aleatoriamente, para que el sistema no dependa de una sola figura. Riesgo: dropout demasiado alto hace difícil aprender.

Early Stopping

Early stopping no cambia la arquitectura ni la función de costo. Observa la pérdida de validación y detiene el entrenamiento cuando deja de mejorar durante cierta cantidad de epochs, llamada paciencia.

Si val_loss no mejora durante p epochs → detener entrenamiento.

Ventaja: simple, eficaz y muy usado. Idea clave: el mejor modelo no siempre es el del último epoch, sino el del mejor punto de validación.

Más datos, data augmentation y simplificación

Una forma muy potente de regularizar es mejorar la información disponible: más datos, datos más diversos o transformaciones realistas. En imágenes se usan rotaciones, recortes o cambios de brillo; en series temporales se pueden usar ventanas; en texto, variaciones controladas.

También se puede regularizar reduciendo la complejidad: menos capas, menos neuronas, menos parámetros o arquitectura más adecuada al problema.

4. Práctica guiada para clase

Escenario A: provocar overfitting

Alta complejidad, bajo L2, bajo dropout y ruido moderado. El objetivo es ver cómo train baja mientras validación empeora.

Escenario B: buscar equilibrio

Complejidad media, L2 moderado, dropout controlado. La validación debería acompañar mejor al entrenamiento.

Escenario C: provocar underfitting

Baja complejidad o regularización excesiva. La red queda demasiado limitada y no aprende bien ni en entrenamiento.

Consigna desarrollada

1
Ejecutá el escenario A y entrená 40 epochs. Anotá cuándo comienza a crecer la brecha entre train y validación.
2
Subí L2 y dropout de forma gradual. Observá si la brecha baja y si la curva ajustada deja de perseguir el ruido.
3
Activá una paciencia baja de early stopping. Identificá visualmente el punto donde conviene detener el entrenamiento.
4
Compará los tres diagnósticos: underfitting, buen ajuste y overfitting. Explicá cuál modelo elegirías y por qué.

5. Conexión con Keras / TensorFlow

Regularización L2 y dropout

from tensorflow.keras import regularizers

model.add(Dense(64, activation='relu',
  kernel_regularizer=regularizers.l2(0.001)))
model.add(Dropout(0.3))

Early stopping

from tensorflow.keras.callbacks import EarlyStopping

early = EarlyStopping(
  monitor='val_loss', patience=8,
  restore_best_weights=True
)
Idea para remarcar: en una práctica real se debe separar train, validation y test. La validación ayuda a elegir hiperparámetros; el test se reserva para una evaluación final honesta.

6. Preguntas de comprensión

Preguntas

  1. ¿Por qué un error de entrenamiento muy bajo puede ser una mala señal?
  2. ¿Qué diferencia conceptual hay entre L1 y L2?
  3. ¿Por qué dropout no se interpreta como “borrar neuronas para siempre”?
  4. ¿Qué significa restaurar los mejores pesos en early stopping?
  5. ¿Cómo detectarías regularización excesiva mirando curvas?

Respuestas esperadas

  1. Porque puede indicar memorización y no generalización.
  2. L2 suaviza pesos grandes; L1 favorece pesos cero y modelos más escasos.
  3. Porque el apagado ocurre durante entrenamiento; en inferencia se usa la red completa.
  4. Volver al epoch con mejor desempeño de validación.
  5. Train y validación permanecen altos: el modelo no tiene capacidad efectiva para aprender.
