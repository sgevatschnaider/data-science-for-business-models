1. Teoría completa: redes recurrentes y memoria temporal

1.1. Por qué una red densa no alcanza

Una red neuronal densa tradicional recibe una entrada y produce una salida sin memoria explícita del pasado. Esto puede funcionar cuando cada observación contiene toda la información necesaria, pero falla cuando el significado depende del orden. En una frase, una palabra puede modificar a otra que aparece muchos términos después; en una serie financiera, el presente puede depender de tendencia, estacionalidad, shocks y rezagos.

Idea central: cuando el orden de los datos cambia el significado, el modelo debe incorporar una representación de contexto.

1.2. RNN simple

Una RNN procesa la secuencia elemento por elemento. En cada paso temporal combina la entrada actual x_t con el estado oculto anterior h_(t-1). El nuevo estado h_t funciona como memoria comprimida de lo visto hasta ese momento.

h_t = tanh(W_xh · x_t + W_hh · h_(t-1) + b_h)
y_t = W_hy · h_t + b_y
ElementoFunción matemáticaLectura didáctica
x_tVector de entrada en el tiempo tLo que el modelo observa ahora.
h_(t-1)Estado oculto previoLa memoria del pasado inmediato y acumulado.
h_tEstado oculto actualizadoEl resumen actual de la secuencia.
W_xhPesos de entradaCómo la entrada modifica la memoria.
W_hhPesos recurrentesCómo el pasado influye en el presente.

1.3. Despliegue temporal y BPTT

La RNN puede imaginarse como una misma celda copiada a lo largo del tiempo. Aunque el diagrama muestre varias celdas, los parámetros son compartidos. El entrenamiento se realiza con Backpropagation Through Time, es decir, el error se propaga hacia atrás a través de los pasos temporales.

x_1 → h_1 → y_1
x_2 → h_2 → y_2
x_3 → h_3 → y_3
x_4 → h_4 → y_4

1.4. Vanishing gradient y exploding gradient

Al retropropagar el error por muchos pasos, el gradiente puede debilitarse o crecer en exceso. Si se debilita, las relaciones lejanas casi no reciben señal de aprendizaje; si crece demasiado, el entrenamiento se vuelve inestable.

Vanishing gradient: el gradiente se vuelve muy pequeño y la red aprende mal dependencias largas.
Exploding gradient: el gradiente crece demasiado y puede generar actualizaciones descontroladas.

1.5. LSTM: memoria regulada

La LSTM incorpora una vía de memoria más estable, llamada estado de celda C_t. A diferencia de una RNN simple, no depende solo de un estado oculto comprimido. Usa compuertas que deciden cuánto olvidar, cuánto guardar y cuánto exponer como salida.

f_t = σ(W_f · [h_(t-1), x_t] + b_f)   // forget gate
i_t = σ(W_i · [h_(t-1), x_t] + b_i)   // input gate
Ĉ_t = tanh(W_c · [h_(t-1), x_t] + b_c) // memoria candidata
C_t = f_t * C_(t-1) + i_t * Ĉ_t       // nueva memoria
o_t = σ(W_o · [h_(t-1), x_t] + b_o)   // output gate
h_t = o_t * tanh(C_t)                 // salida / estado oculto
Forget gateDecide qué parte de la memoria anterior se conserva o se borra.
Input gateDecide qué parte de la nueva información candidata se incorpora.
Output gateDecide qué parte del estado de celda se muestra como estado oculto.

3. Comparación de arquitecturas

ArquitecturaFortalezaLimitaciónUso típico
RNN simpleIntroduce memoria temporal con una estructura relativamente simple.Sufre con dependencias largas por vanishing gradient.Secuencias cortas y explicación introductoria.
LSTMRegula memoria con compuertas y estado de celda.Más parámetros y mayor costo que una RNN simple.Texto, series temporales, señales y secuencias con dependencias largas.
GRUAlternativa más compacta que LSTM, con buen desempeño práctico.Menos explícita conceptualmente que LSTM para explicar memoria separada.Aplicaciones donde se busca eficiencia y buen rendimiento.
TransformersUsan atención para modelar relaciones largas con alta paralelización.Requieren más datos y cómputo; no siempre son la opción más simple.Lenguaje natural, modelos generativos, embeddings y tareas modernas de secuencias.
Mensaje estratégico: las RNN y LSTM siguen siendo muy útiles para comprender la noción de memoria secuencial. Aunque los Transformers dominen muchas tareas actuales, la lógica de estado, contexto y dependencia temporal se entiende con claridad a partir de RNN y LSTM.

5. Errores frecuentes y cómo corregirlos

Error: pensar que una RNN guarda literalmente todo el pasado.
Corrección: guarda una representación comprimida y entrenable.
Error: creer que LSTM elimina por completo el vanishing gradient.
Corrección: lo mitiga mediante una vía de memoria más estable, pero no resuelve todos los problemas.
Error: usar LSTM sin preparar correctamente la forma de entrada.
Corrección: recordar la estructura: muestras, pasos temporales y variables.
Error: comparar modelos solo por exactitud final.
Corrección: analizar longitud de dependencia, estabilidad, costo, interpretabilidad y objetivo de negocio.
