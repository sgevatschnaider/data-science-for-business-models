1. Teoría completa: por qué las CNN son distintas

Volver arriba

Una imagen no es solamente una lista de números. Tiene estructura espacial: píxeles vecinos suelen pertenecer al mismo borde, textura, sombra u objeto. Una red densa tradicional pierde gran parte de esa geometría porque aplana la imagen demasiado pronto. Una CNN conserva la proximidad local y aprende filtros que se desplazan por toda la imagen.

Conectividad localUn filtro mira una región pequeña, por ejemplo 3×3, y no todos los píxeles al mismo tiempo.
Pesos compartidosEl mismo filtro se aplica en muchas posiciones, reduciendo parámetros y detectando patrones en cualquier zona.
Jerarquía visualLas primeras capas detectan bordes; las capas medias, partes; las profundas, conceptos más abstractos.
Idea clave: una CNN no “mira” la imagen completa de una vez. Construye mapas de activación: cada mapa responde a cierto patrón visual aprendido.
ComponenteQué haceIntuición de clase
Filtro / kernelMatriz pequeña de pesos que se desliza sobre la imagen.Funciona como una lupa especializada: una detecta bordes, otra texturas, otra curvas.
ConvoluciónMultiplica región local por el filtro y suma el resultado.Produce un mapa que indica dónde aparece el patrón buscado.
PaddingAgrega borde artificial, generalmente ceros.Evita perder información de los bordes y permite controlar el tamaño de salida.
StrideDefine cuántos píxeles avanza el filtro en cada paso.A mayor stride, menor resolución de salida.
PoolingResume regiones locales, por ejemplo tomando el máximo.Reduce tamaño y conserva la activación más relevante.
Flatten + DenseConvierte mapas en vector y decide la clase final.Integra la evidencia visual aprendida por las capas convolucionales.

9. Guía práctica para clase o Colab

Volver arriba

Después de las simulaciones visuales, conviene llevar a los estudiantes a una implementación mínima. El objetivo no es memorizar código, sino reconocer las piezas: convolución, activación, pooling, flatten y capa densa.

Paso 1
Cargar imágenes. Usar un dataset simple, por ejemplo dígitos, prendas o imágenes pequeñas. Normalizar píxeles a rango 0–1.
Paso 2
Definir arquitectura. Conv2D → ReLU → MaxPooling → Conv2D → ReLU → Flatten → Dense.
Paso 3
Entrenar y evaluar. Separar train/test, medir accuracy y revisar errores de clasificación.
Paso 4
Visualizar activaciones. Mostrar qué mapas se encienden ante bordes, curvas o texturas.

Bloque de código base

import tensorflow as tf
from tensorflow.keras import layers, models

model = models.Sequential([
    layers.Conv2D(32, (3, 3), activation="relu", input_shape=(28, 28, 1)),
    layers.MaxPooling2D((2, 2)),
    layers.Conv2D(64, (3, 3), activation="relu"),
    layers.MaxPooling2D((2, 2)),
    layers.Flatten(),
    layers.Dense(64, activation="relu"),
    layers.Dense(10, activation="softmax")
])

model.compile(optimizer="adam",
              loss="sparse_categorical_crossentropy",
              metrics=["accuracy"])
model.summary()
