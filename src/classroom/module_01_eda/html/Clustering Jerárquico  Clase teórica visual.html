1. Qué problema intenta resolver el clustering jerárquico

En muchos problemas de ciencia de datos tenemos observaciones sin etiquetas previas. Es decir, tenemos datos de clientes, libros, empresas, películas, documentos o transacciones, pero no sabemos de antemano a qué grupo pertenece cada observación.

El clustering jerárquico intenta resolver ese problema construyendo una estructura de agrupamiento progresiva. En lugar de pedir desde el inicio una cantidad fija de clusters, como ocurre en K-Means, el método jerárquico permite observar cómo los datos se agrupan en distintos niveles.

Idea central: el clustering jerárquico no solo responde “qué grupos hay”, sino también “cómo se fueron formando esos grupos”.

2. Intuición geométrica

La idea más simple es imaginar que cada observación es un punto en un plano. Si dos puntos están cerca, el algoritmo interpreta que son parecidos. Si están lejos, interpreta que son diferentes.

Al comienzo, cada punto es un cluster individual. Luego el algoritmo busca los dos puntos o grupos más parecidos y los fusiona. Después vuelve a repetir el procedimiento: busca la próxima fusión más razonable, la registra y continúa hasta que todos los puntos quedan unidos en un único árbol.

Inicio: n observaciones → n clusters individuales
Proceso: fusionar los dos grupos más compatibles
Final: 1 árbol jerárquico completo

La ventaja didáctica es que el proceso puede verse como una historia: primero aparecen pequeñas parejas de puntos muy similares, después subgrupos, luego grupos más grandes y finalmente una estructura completa.

3. Conceptos fundamentales

X

Dato u observación

Es la unidad de análisis. Puede representar un cliente, un libro, una empresa o una transacción. Para aplicar el algoritmo, cada observación debe estar representada por variables comparables.

d

Distancia

Es la medida que permite saber qué tan parecidas o diferentes son dos observaciones. Una distancia baja indica similitud; una distancia alta indica diferencia.

C

Cluster

Es un grupo de observaciones similares. En clustering jerárquico, los clusters se forman mediante fusiones sucesivas.

↔

Fusión

Es el momento en el que dos puntos, dos clusters, o un punto y un cluster se unen para formar un nuevo grupo.

L

Matriz linkage

Es la tabla que registra cada fusión: qué se unió, con qué se unió, a qué distancia y cuántos elementos contiene el nuevo cluster.

T

Dendrograma

Es el árbol visual que representa la matriz linkage. Muestra el orden de las fusiones y la altura a la que ocurren.

4. El algoritmo paso a paso

Paso 1 Cada punto empieza como cluster individual.
Paso 2 Se calculan distancias entre puntos o clusters.
Paso 3 Se elige la fusión más conveniente.
Paso 4 Se registra la fusión en la matriz linkage.
Paso 5 Se construye el dendrograma y se decide el corte.
1

Inicialización

Si tenemos 120 observaciones, el algoritmo comienza con 120 clusters. Cada observación está sola. Todavía no hay grupos grandes ni estructura jerárquica.

2

Cálculo de distancias

El algoritmo mide qué tan lejos está cada observación de las demás. Si hay n observaciones, la cantidad de pares posibles es:

n × (n - 1) / 2

Por ejemplo, con 120 observaciones se calculan 7140 distancias.

3

Primera fusión

El algoritmo busca los dos elementos más compatibles según la distancia y el método de linkage elegido. Si dos puntos están muy cerca, probablemente se fusionen al comienzo.

4

Repetición del proceso

Después de cada fusión, hay un cluster menos. El algoritmo vuelve a calcular qué fusión conviene realizar y repite el proceso hasta que todos los puntos quedan integrados en un único árbol.

5. Qué significa el método de linkage

El método de linkage define cómo se mide la distancia entre clusters cuando los grupos ya tienen más de un punto. Esta decisión es fundamental porque cambia la forma del dendrograma y la estructura final de los clusters.

Método Qué mide Efecto típico
Single linkage La menor distancia entre puntos de ambos clusters. Puede formar cadenas largas.
Complete linkage La mayor distancia entre puntos de ambos clusters. Tiende a formar grupos compactos.
Average linkage El promedio de todas las distancias entre puntos de ambos clusters. Busca una solución intermedia.
Ward La fusión que menos aumenta la variabilidad interna. Genera clusters compactos y equilibrados.
Atención: dos dendrogramas construidos con los mismos datos pueden verse distintos si se cambia el método de linkage.

6. Cómo se interpreta el dendrograma

El dendrograma es la representación visual de las fusiones. En la parte inferior aparecen las observaciones o clusters pequeños. A medida que subimos, vemos cómo esos elementos se van uniendo.

La altura de cada unión indica la distancia de fusión. Una unión baja indica que los elementos eran muy parecidos. Una unión alta indica que el algoritmo está juntando grupos más diferentes.

Fusión baja → grupos parecidos
Fusión alta → grupos diferentes

Para obtener una cantidad concreta de clusters, se traza una línea horizontal de corte. Cada rama que queda separada por debajo de esa línea representa un cluster.

9. Errores comunes al interpretar clustering jerárquico

Creer que el dendrograma decide automáticamente el número de clusters

El dendrograma muestra la estructura de fusiones, pero el analista debe decidir dónde cortar el árbol. La decisión depende de la altura de las fusiones, el objetivo del análisis y la interpretación del problema.

Confundir distancia baja con importancia causal

Que dos observaciones estén cerca no significa que una cause a la otra. Solo significa que son parecidas según las variables incluidas en el análisis.

Ignorar la escala de las variables

Como el método usa distancias, las variables con valores numéricos grandes pueden dominar el resultado. Por eso suele ser necesario estandarizar.

Interpretar todos los colores del dendrograma como verdades definitivas

Los colores ayudan a visualizar posibles clusters, pero la interpretación debe complementarse con conocimiento del problema, métricas y análisis contextual.

10. Guion docente antes de la simulación

“Antes de ver la simulación, necesitamos entender qué problema intenta resolver el clustering jerárquico. Muchas veces tenemos datos sin etiquetas: clientes, libros, empresas, países, documentos o transacciones, pero no sabemos de antemano qué grupos existen.”

“A diferencia de K-Means, acá no empezamos eligiendo K ni colocando centroides. En clustering jerárquico, cada punto empieza como un cluster individual. Luego el algoritmo mide distancias y va fusionando los puntos o grupos más compatibles.”

“Cada fusión queda registrada en una tabla llamada matriz linkage. Esa matriz tiene cuatro datos fundamentales: qué se unió, con qué se unió, a qué distancia ocurrió la fusión y cuántos elementos contiene el nuevo cluster.”

“El dendrograma es el dibujo de esa matriz. Las fusiones bajas representan observaciones muy parecidas; las fusiones altas representan grupos más diferentes. Cuando cortamos el dendrograma, decidimos cuántos clusters queremos conservar.”

Frase de transición hacia la simulación

Ahora que entendimos la lógica teórica, vamos a ver la simulación para observar cómo estas ideas aparecen visualmente: primero los puntos separados, luego las fusiones, después la matriz linkage, el dendrograma y finalmente el corte del árbol.

11. Glosario completo

Concepto Definición desarrollada
Aprendizaje no supervisado Rama del machine learning en la que los datos no tienen etiquetas previas. El objetivo es descubrir patrones, estructuras o grupos internos.
Clustering Técnica que agrupa observaciones similares entre sí. No predice una categoría conocida, sino que propone una organización de los datos.
Clustering jerárquico Método de clustering que construye una jerarquía de grupos. En la versión aglomerativa, comienza con puntos separados y los fusiona progresivamente.
Fusión Unión entre dos puntos, dos clusters, o un punto y un cluster. Cada fusión genera un nuevo cluster.
Matriz linkage Tabla que guarda el historial de fusiones. Cada fila indica los dos elementos fusionados, la distancia de fusión y el tamaño del nuevo cluster.
Dendrograma Gráfico en forma de árbol que representa visualmente las fusiones registradas en la matriz linkage.
Altura de fusión Valor vertical del dendrograma que indica a qué distancia se unieron dos elementos o grupos.
Linkage Criterio que define cómo se mide la distancia entre clusters. Puede ser single, complete, average o Ward.
Ward Método de linkage que elige la fusión que menos aumenta la variabilidad interna del nuevo cluster.
Corte del árbol Decisión que transforma el dendrograma en una cantidad concreta de clusters finales.
