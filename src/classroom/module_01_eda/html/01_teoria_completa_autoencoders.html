Autoencoders: compresión, espacio latente y reconstrucción inteligente

Un autoencoder es una red neuronal que aprende a reconstruir su entrada pasando por una representación interna. Su valor no está en copiar, sino en descubrir una estructura latente que permita conservar lo esencial y descartar lo accesorio.

Idea central: si obligamos al modelo a reconstruir un dato desde una representación limitada o regularizada, la red debe aprender qué información realmente importa.

1. ¿Qué es un autoencoder?

Un autoencoder es una arquitectura neuronal entrenada para producir una salida lo más parecida posible a su entrada. La forma más simple consiste en tomar un vector, una imagen, una señal, una tabla o una secuencia, comprimirla en una representación latente y reconstruirla.

Entrada x
→
Encoder fθ
→
Latente z
→
Decoder gφ
→
Reconstrucción x̂
z = fθ(x)
x̂ = gφ(z)
Objetivo: x̂ ≈ x

La palabra “auto” no significa que se entrene solo mágicamente, sino que la propia entrada funciona como objetivo de aprendizaje. Por eso suele ubicarse dentro del aprendizaje no supervisado o auto-supervisado.

2. Arquitectura: encoder, cuello de botella y decoder

EncoderTransforma el dato original en una representación más compacta o más estructurada.
Espacio latenteContiene variables internas aprendidas. Puede funcionar como embedding del dato.
DecoderReconstruye el dato original usando únicamente la información latente.

El cuello de botella puede ser dimensional, estadístico o regularizado. En un autoencoder subcompleto, el vector latente tiene menor dimensión que la entrada. En uno sobrecompleto, el vector puede ser grande, pero se imponen restricciones para evitar la copia trivial.

TipoRestricciónQué aprende
SubcompletoDimensión latente menor que la entrada.Compresión y factores dominantes.
SparsePocas neuronas activas por ejemplo.Representaciones selectivas e interpretables.
DenoisingEntrada corrupta, objetivo limpio.Estructura robusta frente al ruido.
ContractivePenaliza sensibilidad excesiva del encoder.Representaciones estables ante pequeñas perturbaciones.
VariationalLatente probabilístico con regularización KL.Espacio continuo apto para generación.

3. Entrenamiento y función de pérdida

El entrenamiento minimiza una pérdida de reconstrucción. Para imágenes continuas puede usarse error cuadrático medio. Para datos binarios o píxeles normalizados puede usarse entropía cruzada binaria. Para señales, audio, texto o series temporales se elige una función compatible con el tipo de dato.

L(x, x̂) = ||x - x̂||²
min θ,φ  L(x, gφ(fθ(x)))
Advertencia: reconstruir bien no siempre implica aprender una representación útil. Un modelo demasiado grande puede memorizar o copiar. Por eso importan el bottleneck, la regularización, el ruido, la validación y la evaluación downstream.

4. Espacio latente, manifold y embeddings

El espacio latente es una representación interna aprendida. En datos complejos de alta dimensión, muchos ejemplos reales viven cerca de una variedad de menor dimensión. El autoencoder intenta aprender coordenadas útiles sobre esa variedad.

En imágenes, el latente puede codificar orientación, grosor, curvatura, estilo o iluminación. En datos financieros, puede condensar volatilidad, régimen, correlaciones y riesgo. En texto o embeddings, puede representar tema, tono o estructura semántica.

Lectura geométrica: puntos cercanos en el espacio latente deberían representar datos parecidos.
Lectura práctica: el encoder puede reutilizarse como extractor de features para clustering, clasificación, búsqueda de similitud o anomalías.

7. Errores frecuentes y buenas prácticas

Error 1
Usar una red demasiado grande. Puede copiar la entrada sin aprender estructura. Ajustar dimensión latente, dropout, ruido o penalizaciones.
Error 2
Medir solo la pérdida promedio. Conviene inspeccionar reconstrucciones, distribución de errores y casos extremos.
Error 3
Confundir error alto con anomalía segura. Un error alto indica rareza respecto del entrenamiento, no necesariamente fraude, falla o ataque.
Error 4
No escalar datos tabulares. Variables con mayor escala dominan la pérdida. Normalizar o estandarizar es clave.
Error 5
Interpretar literalmente cada dimensión latente. Algunas dimensiones pueden ser útiles pero no semánticamente claras.
