Entrada textual01 / 11

Tokenización: la frase se convierte en unidades computables

La tokenización toma texto humano y lo convierte en piezas que el modelo puede indexar. En clase se puede mostrar con palabras completas, pero en modelos reales aparecen subpalabras: fragmentos que ayudan a manejar palabras nuevas, nombres propios, tecnicismos y variaciones morfológicas.

El ejemplo se centra en servicios exportables porque permite ver cómo una frase de negocios se parte en unidades que luego serán transformadas en vectores.

"La economía digital transforma servicios exportables" → ["La", "economía", "digital", "transforma", "servicios", "exportables"]
Clave didáctica: tokenizar no significa comprender; significa preparar la frase para que una red neuronal pueda operar sobre ella.

Representación inicial02 / 11

Embeddings: cada token pasa a ser un vector denso

Un embedding representa un token como una lista de números aprendida durante entrenamiento. Palabras con usos parecidos tienden a ubicarse cerca en el espacio vectorial, pero el embedding inicial todavía no resuelve por sí solo una ambigüedad contextual.

Por eso usamos dos ejemplos distintos con la palabra “banco”: uno financiero y otro geográfico. La palabra es la misma, pero el contexto empuja la representación hacia zonas semánticas diferentes.

x_i = Embedding(token_i) "banco" + "crédito" ≠ "banco" + "río"
Clave didáctica: el embedding inicial es una base; la atención posterior lo vuelve contextual.

Orden de la secuencia03 / 11

Codificación posicional: el orden también informa

El Transformer procesa tokens en paralelo. Para no perder el orden, suma o incorpora información posicional. Sin esa señal, una bolsa de palabras tendría los mismos elementos aunque el sentido sintáctico cambiara.

El ejemplo muestra una frase de analítica: el orden “modelo predice ventas” es interpretable; reordenar los mismos tokens puede romper la lectura.

z_i = Embedding(token_i) + Posición_i
Clave didáctica: el modelo necesita saber qué token viene primero, cuál modifica a cuál y qué estructura temporal sostiene la frase.

Proyecciones aprendidas04 / 11

Q, K y V: cada token se mira desde tres funciones

Self-attention no compara palabras directamente. Primero proyecta cada vector en tres espacios: Query, Key y Value. La Query pregunta qué necesita el token activo; la Key ofrece una firma de cada token; el Value aporta contenido que será combinado si recibe peso.

El ejemplo médico separa claramente agente, acción, objeto e instrumento: “La médica revisó la imagen con IA”.

Q = XW_Q K = XW_K V = XW_V
Clave didáctica: Q, K y V no son etiquetas manuales; son transformaciones aprendidas por la red.

Matriz de atención05 / 11

Self-attention: cada token decide a qué otros tokens mirar

La Query del token activo se compara con las Keys de todos los tokens. Los puntajes se normalizan con softmax y se usan para mezclar Values. Así, el vector final del token queda enriquecido por el contexto relevante.

Acá usamos una frase financiera porque “banco” solo se interpreta correctamente si atiende a “aprobó”, “crédito” y “empresa”.

Attention(Q,K,V) = softmax((QKᵀ) / √d_k) V
Clave didáctica: la fila activa de la matriz responde: “¿qué necesita mirar este token para entenderse en esta oración?”.

Lecturas paralelas06 / 11

Multi-head attention: varias formas de leer la misma frase

Una sola matriz de atención sería limitada. Con múltiples cabezas, el modelo puede distribuir funciones: una cabeza mira relaciones semánticas, otra sintaxis, otra proximidad local y otra restricciones causales.

El ejemplo didáctico usa una frase de política pública: cada cabeza ilumina una relación distinta.

MultiHead(Q,K,V) = Concat(head₁, ..., headₕ)W_O
Clave didáctica: las cabezas no son interpretables de forma perfecta, pero ayudan a mostrar que el modelo aprende patrones complementarios.

Comprensión contextual07 / 11

Encoder: construir una representación rica de la entrada

El encoder combina self-attention, conexión residual, normalización y una red feed-forward. Su objetivo principal es producir representaciones contextuales útiles para clasificación, búsqueda semántica, extracción de entidades o recuperación de información.

El ejemplo usa atención al cliente: el encoder debe entender intención, entidad y urgencia.

Y = LayerNorm(X + SelfAttention(X)) Z = LayerNorm(Y + FFN(Y))
Clave didáctica: el encoder no genera necesariamente texto; primero comprende y estructura la entrada.

Generación autoregresiva08 / 11

Decoder: generar mirando solo el pasado disponible

En generación, el modelo predice el siguiente token paso a paso. Para evitar que “vea el futuro” durante entrenamiento, se usa una máscara causal: cada posición solo puede atender a posiciones anteriores o a sí misma.

El ejemplo se adapta a redacción: “La IA genera un informe...” y el modelo debe continuar sin acceder a tokens futuros.

masked_score(i,j) = -∞ si j > i
Clave didáctica: la máscara causal explica por qué los modelos generativos producen texto de izquierda a derecha.

Entrada condicionante09 / 11

Cross-attention: el decoder consulta la salida del encoder

En tareas encoder-decoder, como traducción o resumen, el decoder no solo mira lo que ya generó. También consulta la representación de la entrada codificada por el encoder. Allí las Queries vienen del decoder y las Keys/Values vienen del encoder.

El ejemplo traduce una frase técnica breve: “The model predicts sales” hacia “El modelo predice ventas”.

CrossAttention(Q_decoder, K_encoder, V_encoder)
Clave didáctica: cross-attention permite que una salida se mantenga alineada con una entrada original.

Distribución final10 / 11

Generación: el próximo token sale de una distribución

El Transformer no escribe una palabra directamente. Proyecta el vector final al tamaño del vocabulario y produce una distribución de probabilidades. Después, una estrategia de decodificación elige o muestrea el próximo token.

El ejemplo médico usa una continuación posible después de “El diagnóstico sugiere...”.

p(token_siguiente) = softmax((Wh + b) / T)
Clave didáctica: la temperatura baja concentra la salida; la temperatura alta aumenta diversidad y riesgo.

Uso responsable11 / 11

Lectura crítica: atención no equivale a explicación total

Las visualizaciones de atención son muy útiles para enseñar, pero no deben venderse como explicación perfecta del razonamiento del modelo. Un peso alto indica influencia en la combinación de Values; no demuestra por sí solo intención, causalidad o comprensión humana.

Para evaluar un Transformer conviene mirar ejemplos, errores, sesgos, datos de entrenamiento, métricas de tarea, pruebas fuera de distribución y comportamiento ante instrucciones ambiguas.

visualización + evaluación + análisis de errores + criterios de dominio
Clave didáctica: la matriz de atención es una pista interpretativa, no una auditoría completa del modelo.

Checklist de interpretación

Usá estos criterios para cerrar la clase con mirada técnica y responsable.

cierre
Ambigüedad

Una misma palabra puede tener sentidos distintos según el contexto.

Alucinación

Una salida probable no siempre es verdadera o verificable.

Sesgo

Los patrones aprendidos reflejan datos, incentivos y distribución de entrenamiento.

Contexto

El modelo trabaja dentro de una ventana limitada y puede perder información distante.

Evaluación

La calidad debe medirse con tareas, ejemplos y criterios de dominio.

Interpretabilidad

Atención ayuda a visualizar, pero no agota el comportamiento interno.

Cierre sugerido: “Un Transformer es poderoso porque calcula relaciones contextuales; usarlo bien exige comprender tanto su mecánica como sus límites”.
